# SEO执行总规 v1（LocalVRAM）

生效日期：2026-02-26  
状态：执行中（规划版）

## 1. 总目标（90天）

1. 收录页数提升 40%+。
2. P0/P1 关键词进入 Top10 的数量稳定增长。
3. 工具页 SEO 流量占比提升到 35%+。
4. 工具页到转化页 CTR 提升到 8%+。

## 2. 收录策略（双轨并行）

### 2.1 轨道A：Indexing API 点火（非全站主策略）

白名单：

1. `https://localvram.com/en/updates/`
2. `https://localvram.com/en/benchmarks/changelog/`

约束：

1. 每日最多 2 次推送（全站总量）。
2. 同一 URL 24 小时最多 1 次。
3. 只有“实质更新”才推送（内容 hash 变化）。
4. 非白名单页面禁止推送（模型页、博客页、错误页、指南页）。

失败与熔断：

1. 网络失败重试：`5s/10s/20s`。
2. 429：等待 60s 后单次重试。
3. 连续高失败率触发 7 天熔断，回退到自然收录机制。

### 2.2 轨道B：自然收录（主策略）

对象：

1. `/en/models/*`
2. `/en/blog/*`
3. `/en/errors/*`
4. `/en/guides/*`
5. `/en/hardware/*`

机制：

1. `sitemap + Silo内链 + 分批曝光 + 内容厚度`。
2. Daily Updates 每天曝光 5-10 个模型页。
3. 曝光优先级：最新模型优先，其次一个月前模型。

## 3. `/all-models` 爬虫入口规划

### 3.1 URL结构

1. 主入口：`/en/models/all/`
2. 分页：`/en/models/all/page/N/`

### 3.2 页面构成

1. 入口说明区（索引用途）。
2. 分类导航（size/scenario/license）。
3. 全量列表区（分页，静态可抓取链接）。
4. 分页导航与回流入口（模型页、工具页）。

### 3.3 规则

1. 每页建议 100 条。
2. 排序：最新模型优先，再一个月前模型，再常规稳定页。
3. 每页 self-canonical，禁止参数化重复 URL。
4. 空分页返回 404。

## 4. 模型页内容厚度标准

### 4.1 必须项（全部满足）

1. 独特结论段（80-150字，非模板句）。
2. 数据来源标识（measured/estimated）+ 时间戳。
3. 可执行建议 2 条（本地/云）。
4. 至少 1 个对比点（同档位或同家族）。
5. 至少 3 条深链（group/工具/错误或指南）。

### 4.2 打分规则（1-5）

1. `5分`：必须项全满足 + 2项以上加分项。
2. `4分`：必须项全满足 + 1项加分项。
3. `3分`：必须项全满足（上线最低线）。
4. `2分`：缺 1 个必须项。
5. `1分`：缺 2 个以上必须项（禁止曝光）。

### 4.3 曝光准入

1. 最新模型页：评分 `>=3` 才可曝光。
2. 一个月前模型页：评分 `>=4` 才可入补收录队列。

## 5. 14天曝光执行规则（当前版本）

1. 每天 8 个 URL。
2. 配比：`5个最新 + 2个一个月前 + 1个兜底页`。
3. `qwen3.5:35b` 对应页面固定每日第1位。
4. 同页 7 天内不重复主曝光（除非有新实测更新）。

## 6. 监控与告警

### 6.1 每日字段

1. `discovered/crawled/indexed`
2. `days_since_publish`
3. `impressions/clicks/ctr/avg_position`
4. `internal_links_count`
5. `content_thickness_score`
6. `is_in_daily_updates_today`

### 6.2 阈值（周维度）

1. 收录率 `<50%`：红色告警。
2. 最新模型页平均收录时延 `>10天`：红色告警。
3. 工具页 CTR `<2%`：红色告警。
4. 已发现未收录占比 `>40%`：红色告警。

### 6.3 自动动作

1. 7天未收录：补 1 条高权重内链 + 次日复曝一次。
2. 连续 3 天收录率下降：暂停扩量，先修内容厚度与内链。
3. 404 异常增长：立即排查死链与重定向。

## 7. 执行节奏

### 7.1 每日（10分钟）

1. 复盘昨日 8 个曝光页抓取/收录状态。
2. 替换连续 3 天无进展页面。
3. 更新当日 8 页曝光池（按优先级规则）。

### 7.2 每周（30分钟）

1. 看板复盘（收录率、时延、展示、点击）。
2. 输出“有展示无排名”与“有排名无点击”双清单。
3. 锁定下周关键词与页面池。

## 8. 角色分工（建议）

1. SEO Owner：维护曝光池与复盘结论。
2. 内容Owner：保证模型页厚度评分达标。
3. 工程Owner：保障 sitemap、索引页、数据更新链路稳定。
4. 运营Owner：社区分发与 UTM 归因复盘。
