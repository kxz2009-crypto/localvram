## 一、项目概述

### 1.1 项目定位

**一句话定位（英文）**：
> "Ollama Verified: Real 3090 & Apple Silicon Benchmarks — Run or Rent in One Click."

**一句话定位（中文）**：
> "Ollama 真机验证中心 — 用真实数据帮你在 60 秒内选对模型"

### 1.2 核心价值主张

本项目不是一个普通的内容聚合站，而是一个**数据驱动的决策引擎**。核心差异化：

| 传统聚合站 | LocalVRAM |
|-----------|-----------|
| 只有安装命令 + 简介 | 真机验证数据 + 可复现基准 |
| 估算 VRAM 需求 | 实测显存占用 + 上下文膨胀曲线 |
| 静态内容，更新慢 | 每周验证 + 自动更新 |
| 无法帮助决策 | 一键推荐最优方案 |
| 无差异化护城河 | 5大护城河构建壁垒 |

### 1.3 五大核心护城河


### 4.2 模块详细规划

#### 模块一：VRAM 智能预测器 (Fit Checker)

**功能描述**：
用户输入硬件配置，系统推荐可运行的模型及最优量化方案。

**输入参数**：
- GPU 型号 / VRAM 大小
- 系统内存（CPU offload 场景）
- 目标用途（对话/RAG/编程/多模态）
- 期望上下文长度

**输出结果**：
- ✅ 可运行模型列表
- 推荐量化等级
- 预估性能区间（tok/s）
- 显存水位图（权重/上下文/KV cache 分层）
- 每月电费预估 + 本地/云 GPU 盈亏点
- 一键跳转：换小模型 / 换量化 / 上云 GPU

**交互增强（UI 参考补充）**：
- 三张输入卡片使用半透明磨砂风格（硬件/模型/场景）
- 显存超限时卡片边缘触发红色脉冲警示
- 结果区使用动态显存水位计，分层展示系统占用/模型权重/KV Cache

**目标**：保证“每天更新”的内容同时满足高质量、SEO 可收录、可变现三项要求。

**自动化分级**：

| 内容类型 | 更新方式 | 审核方式 |
|---------|---------|---------|
| 价格/版本/日志类数据页 | 全自动 | 抽检 |
| 错误库与兼容矩阵增量 | 半自动 | Agent 生成 + 人工 3 分钟复核 |
| 高转化页面（云 GPU/硬件推荐） | 半自动 | 必须人工终审 |

**每日流水线（建议 UTC 00:30 触发）**：
1. 抓取数据：模型元数据、验证日志、错误频次、Search Console 查询词
2. 机会评分：`机会分 = 搜索意图分 × 商业意图分 × 数据新鲜度缺口`
3. 自动选题：产出当日 `1 新增 + 2 更新` 页面清单
4. 内容生成：按模板生成正文、FAQ、JSON-LD、内部链接
5. 变现绑定：按 `3.12 页面-场景-变现映射速查表` 自动插入 CTA
6. 质量闸门：通过后才允许发布
7. 自动构建部署：推送 GitHub -> Cloudflare Pages
8. 发布后回收：记录 CTR/停留/转化，下次更新回灌评分模型

**质量闸门（必须全部通过）**：
- `Fact Check`：数据字段与日志一致，无虚构跑分
- `SEO Check`：标题、H1、描述、Schema、内链完整
- `Intent Check`：页面内容与关键词搜索意图一致
- `Monetization Check`：至少 1 个主 CTA + 1 个备选 CTA
- `UX Check`：移动端可读，首屏 3 秒内看到结论

**建议新增配置文件**：
- `src/data/content-opportunities.json`：每日选题池与评分
- `src/data/cta-rules.json`：场景到 CTA 的映射规则
- `scripts/daily-content-agent.py`：自动选题 + 生成 + 提交
- `scripts/quality-gate.py`：质量闸门检查
